# API Testing Project – Manual & Postman Testing

This project demonstrates my hands-on experience in API testing using Postman.  
I tested multiple REST endpoints of a demo e-commerce API to validate functionality, response structure, performance, and error handling.

Scope of Testing:
• Functional validation of GET endpoints (Products, Users, Carts)
• Response schema and data type verification using Postman test scripts
• Status code validation (200, 400, 404 scenarios)
• Boundary and negative test cases (invalid IDs, incorrect inputs)
• Response time checks and basic performance validation

Deliverables included:
• API Test Scenarios & Test Cases (Excel)
• API Bug Reports
• Postman test scripts for response validation
• Execution results documentation

Tools & Skills Used:
Postman | REST API Testing | JSON Validation | Test Case Design | Bug Reporting

This project helped me understand real-world API validation workflows and strengthened my ability to analyze backend behavior and data integrity.
